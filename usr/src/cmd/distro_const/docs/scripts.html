Customizing the Finalizer Scripts

The Distribution Constructor application includes the following default scripts in the /usr/share/distro_const directory:

  • slim_cd/post_bootroot_pkg_image_mod

  • slim_cd/pre_bootroot_pkg_image_mod

  • utils/bootroot_initialize.py

  • utils/bootroot_configure

  • utils/bootroot_archive

  • utils/create_iso

  • utils/create_usb

These scripts are already specified in the finalizer section of the default manifest. You can use these default scripts without editing them.

Alternately, you can provide your own scripts and add them to the manifest.

If you choose to add or revise the scripts, note the following characteristics of the finalizer scripts:

  • Scripts can be Python programs, shell scripts or binaries.

  • Scripts are executed in the order that they are listed in the manifest.

  • By default, the Distribution Constructor passes three arguments to any scripts that are executed. These arguments are as follows:

    Server socket file name

    Specifies the server socket that is used for accessing the manifest data

    Package image area path

    Specifies the path to the area where the image is created.

    /tmp directory

    Specifies the path to the /tmp directory for temporary files.

    The arguments are passed into the scripts by tagging them with the <argslist> tag. Other arguments can be passed into scripts by specifying them as key-value pairs. Stdout and stderr can also be specified.

    Note - Spaces used for formatting XML are also passed into the scripts. In your scripts, make sure that the script strips spaces.
