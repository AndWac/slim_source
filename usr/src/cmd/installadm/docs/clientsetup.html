How to Administer an Automated Install Client

The automated installer application enables multiple client systems to access the available installation blueprints that are provided by existing OSInstall services. Client setup is not required. You need only boot the client to initiate service discovery.

Default Client Setup

The installadm create-service command automatically sets up default client specifications for the service that is created, including a default service name and image path for that services' clients.

When the client boots, the default automated installer process is as follows:

  • The client system boots.

  • The client finds its own IP address in the DHCP server, along with the boot file information from the DHCP macro.

  • The boot file points to the pxegrub function, and GRUB boots.

  • The OpenSolaris kernel boots by using the menu.lst information.

  • Additional files are retrieved from the net image by using the menu.lst location.

  • The client retrieves the automated installer's manifest file by using the mDNS service.

  • The automated installer program begins by using the installation specifications from the manifest file. The installer uses these specifications to complete the installation on the client.

Custom Client Setup

The system administrator can use the installadm create-client utility to customize the default client configurations and to limit the services, or to explicitly specify a service name and image path that a client can discover.

In order to create custom client specifications, the administrator must provide the following information:

  • MAC address for the client

  • Path of the net image to be used by the automated installer for this client

  • Name of the install service for the client to use for installation

  • Optionally, you can also specify boot properties and a transport protocol.

Use the following command to provide custom client settings.

installadm create-client [-b <boot_options> ("<property>=<value>,...")] [-P protocol] [-e macaddr] [-t imagepath] [-n svcname]

This command has the following options:

  • -e macaddr – Required: Specifies the MAC address for client.

  • -t imagepath – Required: Specifies the path of the net image to be used with the automated installer.

  • -n svcname – Required: Specifies the service for the client to use for installation.

  • -b <boot_options> ("<property>=<value>,...") – Set one or more property values in the client-specific menu.lst file in /tftpboot. Use this option to set boot properties that are specific to this client. This option can accept multiple property=value pairs.

  • -P protocol – Specifies the transport protocol (HTTP or NFS) for accessing remote archive data. HTTP is the default.
